# Next.js Playground

> **Next.js Playground.** This is my first Next.js project, built to master the App Router, client-side state management, and modern UI patterns.

---

### 🛠 Tech Stack

- **Framework:** Next.js (App Router)
- **Language:** TypeScript
- **State Management:** Zustand
- **Styling:** Tailwind CSS

---

### ✨ Features Implemented

- **Next.js Routing:** Built using the App Router (`/`, `/login`, `/dashboard`).
- **Dummy Authentication:** A secure-feeling flow using **Zustand** to manage login state (persisted locally).
- **Protected Routes:** Dashboard access is guarded and redirects unauthorized users to login.
- **Smart UI States:**
  - Custom `loading.tsx` with a CSS spinner for better UX during navigation.
  - Custom `not-found.tsx` for clean 404 error handling.
- **Layout Architecture:** Utilizes nested layouts to manage global and dashboard-specific navigation.
- **Responsive Design:** Fully styled with Tailwind CSS for mobile and desktop.

---

### 🚀 About the Build

This project serves as my foundational experience with Next.js. The focus was on learning the architecture of the App Router, separating Server vs. Client components (the "use client" boundary), and managing global state effectively.
